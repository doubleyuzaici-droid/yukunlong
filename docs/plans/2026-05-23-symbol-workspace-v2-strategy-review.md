# Symbol Workspace V2 — 投研策略层面深度审查

> **视角**：资深 buy-side 研究员 / 组合经理（PM）。
> **关注点**：不是"显示什么数据"（之前的审查已覆盖可读性/文案/UX/数据接入），而是**"工作台如何帮人形成投资判断"** —— 投研方法论层面的盲区。
> **前置**：decision 态-动作一致性（reduce 态不回退到"做多"）已修复，方向正确。本文档不再重复。

---

## 0. 一句话诊断

V2 当前是一个**优秀的"单标的、单时点、空仓视角的建仓决策"工具**。但真实投研有四个维度它还没覆盖：

```
            当前 V2 覆盖              投研真实需要
时点         单时点快照        →      信号在时间轴上的衰减与有效性
持仓         隐含空仓建仓      →      空仓 / 持有盈 / 持有亏 三态决策
策略         单一共振信号      →      多策略交叉 + regime 有效性
维度         技术/估值分离     →      好公司×好价格×好时机 三维合一
组合         孤立看一只        →      与现有持仓的相关性和集中度
```

下面 8 个问题，按对**决策质量**的影响排序。

---

## 1. 🔴 根本定位局限：只服务"建仓决策"，不服务"持仓管理"

### 问题

V2 所有输出都隐含一个假设：**用户当前空仓，在决定要不要买**。证据：
- 风险预算计算器 [RiskBudgetCalc](TradingAgents/frontend/src/pages/symbol/pulse/RiskBudgetCalc.tsx) 从 0 开始算"建议手数"
- 推荐动作"等待回踩 ¥1,580 再评估买点" —— 是开仓语言
- Hero / Bull-Falsify / 信号都是"该不该进"的框架

但真实场景里，研究员/PM 打开一只票时，**组合里常常已经有仓位**。同一份数据，对三类持仓者意义完全不同：

| 持仓状态 | 用户真正想知道的 | 当前 V2 的回答 |
|---|---|---|
| 空仓 | 要不要建仓？建多少？ | ✅ 回答了 |
| 持有盈利（+15%） | 止盈一部分？加仓？还是持有？ | ❌ 没有 |
| 持有亏损（-12%） | 补仓摊低？割肉止损？还是扛？ | ❌ 没有 |

风险预算给"减仓预警"的用户算"建议建仓 100 股"——逻辑是错位的。

### 落地形态

1. Header 或 ContextRail 加一个**持仓输入**：`成本价 / 持仓数量 / 占组合比`（可选，localStorage 持久化）
2. 一旦有持仓，整个工作台切换到**持仓管理视角**：
   - 风险预算计算器改算"当前已持 X，建议调整到 Y"（加仓/减仓的增量）
   - Hero 推荐动作从"建仓语言"切到"持仓语言"：浮盈时"目标价 ¥1,820 前持有，跌破 ¥1,560 止盈一半"
   - 新增"持仓健康度"：当前浮盈/浮亏、距止损距离、距目标价距离
3. 这是 V2 **从"选股工具"升级到"投研+持仓一体"的关键一跃**，影响所有 tab。

**投研价值**：极高 —— 这是当前最大的方法论盲区。

## 2. 🔴 评分的"含金量"未与市场状态(regime)挂钩

### 问题

共振策略本质是**趋势跟踪型**。趋势市里它如鱼得水，震荡市里它反复假信号、来回打脸。但当前：

- 后验统计 [backtest-summary](TradingAgents/tradingagents/api/professional_routes.py) 是**全样本聚合**：48 个样本，胜率 62.5%
- 这个 62.5% 混合了 2024 牛市、2025 震荡、不同 regime 的样本
- 同一个 0.76 评分，在趋势市可能值 72% 胜率，在震荡市可能只值 38%

**用户看到"评分 76、胜率 62.5%"会过度信任**，因为它没告诉你"当前是什么市场状态、该策略在这种状态下历史表现如何"。

### 落地形态

1. 后端 backtest-summary 增加 **regime 切分**：
   ```
   全样本胜率 62.5% (n=48)
   ├─ 趋势市 72.0% (n=25)   ← 当前所处
   ├─ 震荡市 41.7% (n=12)
   └─ 下跌市 50.0% (n=11)
   ```
   regime 可以用现有 `market_regime` 字段（signal_log / event_return 都已有这列）
2. Hero 评分旁边加一句 regime 提示：
   > 当前市场状态：**震荡市** · 该策略在震荡市历史胜率仅 42%，本次评分谨慎参考
3. 这把"评分可信度"从"因子完整度"（数据维度）升级到"策略-环境匹配度"（方法论维度）。

**投研价值**：极高 —— 这是评分可信度的根本，比"因子完整度 82%"重要得多。

---

## 3. 🟠 估值 × 技术 × 时机 三维割裂

### 问题

投资的本质是 **好公司（quality）× 好价格（value）× 好时机（timing）** 三维合一。当前 V2：
- 技术信号在 Tab 1/2（timing）
- 估值百分位在 Tab 3（value）
- 财务质量在 Tab 3（quality）

**三者分离，互不通气**。最危险的盲区：

> 技术面买入信号 0.76（金叉、放量）+ 估值处于历史 **P95（极贵）** = 最容易套人的"高位金叉"

当前 Tab 1 的买入信号**不知道** Tab 3 的估值已经在历史顶部。研究员如果只看 Tab 1 就进场，可能正好接盘。

### 落地形态

1. 把估值分位作为信号的**折扣因子**注入 Hero：
   - 估值 ≤ P30 → 信号"含金量加成"
   - 估值 ≥ P80 → 信号旁标红："技术面看多，但估值处于历史 P95，注意高位风险"
2. 综合评分可以引入一个"估值调整后评分"：`technical_score × valuation_discount`
3. 三维合一的极简可视化（Tab 1 顶部）：
   ```
   质量 ●●●●○  好公司（ROE 18%，现金流稳）
   估值 ●○○○○  偏贵（PE 历史 P82）
   时机 ●●●○○  趋势在但动能衰减
   → 综合："好公司贵了，等回调，不追高"
   ```

**投研价值**：高 —— 防止"高位金叉"这个最经典的散户陷阱。

## 4. 🟠 Alpha 来源缺失：一致预期差 + 预期修正方向

### 问题

超额收益（alpha）的真正来源是 **市场预期与未来现实的差**，不是历史增速。当前 V2：
- 展示"Q1 净利 +21.7% 超市场一致预期" —— "超预期"是**定性**的，没量化
- 没有"实际 EPS vs 一致预期 EPS"的偏离度（beat/miss 幅度 %）
- 没有"一致预期是在**上修还是下修**" —— 这个方向往往比绝对值更重要

举例：两只票都是"PE 26 倍"，但：
- A：卖方一致预期连续 3 个月上修，目标价从 1600 提到 1820
- B：一致预期连续下修，目标价从 1900 砍到 1700

A 和 B 的投资含义天差地别，但当前 V2 都显示成"PE 26"。

### 落地形态

1. 数据源已确认可行（见 remaining-feasibility §2）：`stock_research_report_em` 聚合 90 日研报
2. Tab 3 "卖方观点"面板：
   ```
   一致预期目标价   ¥1,820   +12.1% 空间    样本 18 家
   预期修正方向     ↗ 近 3 月上修 5 次       ← 关键
   最新业绩 vs 预期 实际 +21.7% / 预期 +18% → beat +3.7pp
   ```
3. "预期修正方向"应该进入 Hero 的支撑理由列表。

**投研价值**：高 —— 这是 buy-side 区别于散户的核心方法论。

## 5. 🟠 资金面只有"量"，没有"质"

### 问题

当前北向、主力资金展示的是**流入流出额**（+2.3 亿、+12.4 亿累计）。但专业投研问的是**资金的质量**：

- 这是**聪明钱领先建仓**，还是**散户追高**？
- 量价配合：放量上涨（健康）/ 放量滞涨（出货）/ 缩量上涨（惜售，待变盘）
- 北向在**高位流入** vs **低位流入** —— 同样金额，含义相反
- 主力净流入但股价不涨 = 可能在对倒诱多

V2 现在只会说"主力 5 日净流入 2.3 亿"，但不会判断这 2.3 亿是好是坏。

### 落地形态

1. 用现有 daily_bars（量价）+ fund_flow_daily（资金）做**资金质量标签**：
   - "放量突破，主力流入与价格同向（健康）"
   - "缩量上涨，惜售待变盘"
   - "高位放量滞涨，警惕出货"
2. 在指标矩阵的"资金"因子旁边，从纯数字升级到"数字 + 质量判断"
3. 北向流入要结合价格位置（用 §价格相对位置条 已有的 P68）：高位流入标黄、低位流入标绿。

**投研价值**：中-高 —— 把资金面从"看数字"升级到"看意图"。

## 6. 🟡 信号的时效与衰减

### 问题

一个 3 天前生成的"买入观察"信号，今天 K 线已经涨了 5%，**这个信号今天还作数吗**？

当前 [SignalDetail](TradingAgents/frontend/src/pages/symbol/playbook/PlaybookTab.tsx) 展示的是**信号生成时的快照**，不告诉用户：
- 信号生成至今过了几天（新鲜度）
- 当前价格是否还在信号假设的"有效区间"内
- 信号是否已经被后续行情证伪/兑现

研究员可能基于一个已经失效的信号做决策。

### 落地形态

1. SignalDetail 加"信号状态"行：
   ```
   生成于 3 天前 · 入场假设 ¥1,580 · 当前 ¥1,623 (已偏离 +2.7%)
   状态：⚠ 已偏离入场区间，建议重新评估而非追价
   ```
2. 信号超过 N 天 / 价格偏离入场区 X% → 自动标"已过期，仅供历史参考"
3. Bull/Falsify 的 falsify 条件如果已触发，整个信号应该置灰。

**投研价值**：中 —— 防止用陈旧信号做新决策。

## 7. 🟡 单一策略 = 单点故障，缺多视角交叉

### 问题

目前工作台只有**一个共振策略信号源**。专业投研的纪律是**多个独立视角交叉验证**：
- 趋势跟踪说：买（追涨）
- 均值回归说：卖（超买回归）
- 估值说：贵了
- 动量说：强者恒强

它们**一致时信号最强，冲突时恰恰是最需要人工判断的地方**。单一策略的风险：
- 单点故障（这个策略失灵就全盘失灵）
- 容易过拟合（一套参数刷一个漂亮回测）
- 给用户虚假的确定性

### 落地形态

1. 不必真的跑多个策略引擎（工作量大）。可以先做**轻量的多视角标签**：
   ```
   共振策略   买入观察 (0.76)
   均值回归   ⚠ RSI 67 偏超买，短期回归压力
   相对强弱   强于行业 +3%、强于大盘 +5%
   估值锚     PE 历史 P82，偏贵
   → 多空分歧：趋势看多 vs 估值/动能看空，等回调
   ```
2. 当多视角一致 → 信号加星；冲突 → 明确提示"分歧，需人工裁决"。

**投研价值**：中 —— 把"单一策略的确定性幻觉"换成"诚实的多空分歧"。

---

## 8. 🟡 孤立看一只票，缺组合关联视角

### 问题

单股工作台**完全孤立**地看一只标的。但 buy-side 的建仓决策从来不是孤立的：
- 买茅台时，组合里是否已经重仓五粮液、泸州老窖？（**行业过度集中**）
- 该标的与现有持仓的**相关性**（买了只跟现有持仓 0.9 相关的票 = 没分散，只是加杠杆）
- 对组合 **beta / 行业暴露**的边际影响

当前 V2 给"建议仓位 8.1%"，但不知道组合里白酒已经占了 25%，这 8.1% 会让白酒到 33% —— 这是组合层面的风险。

### 落地形态

1. 这个依赖"用户组合"数据，是较重的功能。轻量起步：
   - 用 watchlist + 持仓输入（见 §1）做一个极简的"行业集中度"提示
   - "你的关注/持仓中已有 3 只白酒，再建仓将使白酒暴露偏高"
2. 完整版需要组合管理模块，超出单股工作台范围，可作为长期方向。

**投研价值**：中（buy-side 高，个人投资者中）—— 但工作量也最大，建议放后。

## 9. 补充：后验统计的可信度标注（轻量但重要）

当前后验"48 样本胜率 62.5% 质量 A"已经有 sample_quality 分级（好），但还缺投研严谨性：
- **置信区间**：48 样本的 62.5% 胜率，95% 置信区间可能是 [48%, 76%] —— 区间这么宽，说明统计上和"掷硬币"区分度不高
- **in-sample / out-of-sample 区分**：这些样本是不是就是用来调参的那批？（过拟合风险）
- **是否经历完整牛熊**：只有牛市样本的胜率没有意义

**落地**：胜率旁加置信区间 + "样本未跨越完整牛熊周期"的警告标。

**投研价值**：中 —— 防止用户把过拟合的回测当真理。

## 10. Top 5 优先级（按对决策质量的影响 / 工作量）

| 优先级 | 项目 | 投研价值 | 工作量 | 数据源 |
|---|---|---|---|---|
| 🔥 1 | **持仓状态感知**（空仓/持盈/持亏三态）| 极高 | 后 1d + 前 2d | 用户输入，无外部依赖 |
| 🔥 2 | **regime-conditioned 胜率**（当前市场状态下该策略历史有效性）| 极高 | 后 1d + 前 0.5d | 现有 market_regime 字段 |
| 🌟 3 | **估值×技术×时机三维合一 + 高位金叉警告** | 高 | 后 0.5d + 前 1d | 现有 valuation-percentile |
| 🌟 4 | **一致预期差 + 预期修正方向** | 高 | 后 1.5d + 前 0.5d | stock_research_report_em（已验证）|
| 🌟 5 | **资金质量标签**（量价配合 + 高低位流入）| 中-高 | 后 0.8d + 前 0.3d | 现有 daily_bars + fund_flow_daily |

**第一批强烈推荐**：#1 持仓状态感知 + #2 regime 胜率。
- #1 是定位升级（从选股工具到投研+持仓一体），#2 是可信度根基。
- 两者合计约 4.5 天，都不需要新数据源，能把 V2 从"漂亮的建仓工具"提升到"专业的投研决策系统"。

## 11. 反清单：投研策略上**不该**做的

❌ **不要堆更多技术指标**
- 已经有趋势/动能/量能/大盘/资金 5 因子 + RSI/KDJ/MACD/BOLL
- 再加 CCI/威廉/DMI/OBV... 是"指标通货膨胀"，只会稀释信号
- 投研价值在"少而精的多视角"，不在"多而杂的同质指标"

❌ **不要追求"精确的买卖点预测"**
- "目标价 ¥1,820.50" 这种精确度是伪科学
- 应该给区间 + 概率，而不是单点预测

❌ **不要把回测胜率当"承诺"**
- 历史 62.5% 不代表下一次 62.5%
- 永远配可信度区间 + regime 条件 + "不构成投资建议"

❌ **不要做高频/日内策略**
- 这是研究工作台，定位是日线级别的投研决策
- 日内 tick 级是另一个产品

❌ **不要为了"全面"而把组合管理硬塞进单股工作台**
- §8 组合关联性虽有价值，但完整的组合管理是独立模块
- 单股工作台只做"轻量集中度提示"即可，不要变成四不像

---

## 12. 整体判断

V2 在**信息呈现层**已经做到 9/10（色彩、文案、数据、交互都到位）。

但在**投研方法论层**，它目前还停留在"**单时点 × 空仓 × 单策略 × 单维度**"的建仓决策工具。要成为真正的"专业投研决策系统"，关键的跃迁是这四个"维度补全"：

1. **时间维度**：信号有衰减、有时效（§6）
2. **持仓维度**：区分空仓/持盈/持亏（§1）← 最大盲区
3. **策略维度**：regime 有效性 + 多视角交叉（§2 §7）
4. **空间维度**：估值×技术×时机三维 + 组合关联（§3 §8）

完成 Top 2（持仓感知 + regime 胜率）后，V2 的投研专业度会有质的提升 —— 从"告诉你这只票现在的信号"升级到"在你的持仓和当前市场环境下，这个信号对你意味着什么"。
