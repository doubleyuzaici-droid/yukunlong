# 个股分析完整度体系设计

## 背景

当前个股工作台已经能展示行情、K 线、因子、基本面、新闻、信号审查和历史表现，但这些模块依赖的数据表分散在不同接口里。以快手 `01024.HK` 为例，日线和技术因子已覆盖，但主数据、基本面、真实新闻证据、资金流、信号审查和后验归因都有缺口。用户需要的不只是页面空态，而是一套能持续发现、解释和推动补数的完整度体系。

## 目标

建立通用的 `analysis-readiness` 诊断能力，让任意标的都能得到统一的分析完整度评分、缺口解释和下一步行动建议。快手作为第一只样板标的，但实现必须适用于 A 股、港股、美股和指数。

## 范围

第一阶段聚焦“诊断和展示”：

- 后端新增个股分析完整度接口。
- 对主数据、行情、技术因子、基本面、新闻证据、资金流、策略信号、Agent 审查、后验归因、交易规则进行统一评分。
- 识别占位新闻，例如 `No news found for ...`，避免把无效数据当成有效证据。
- 个股工作台展示完整度摘要、阻断项、警告项和下一步建议。
- 保持现有数据同步和策略逻辑不破坏。

后续阶段再补数据同步：

- security master 自动补全。
- 港股基本面和业务分部数据。
- 港股南向/沽空/成交额异常数据。
- 历史 Agent 报告结构化入库。
- 策略、回测和交易规则统一使用同一份 lot size。

## 数据模型

接口返回一个轻量诊断对象：

```json
{
  "symbol": "01024.HK",
  "date": "2026-05-17",
  "score": 0.42,
  "level": "partial",
  "summary": {
    "ready_count": 2,
    "warn_count": 6,
    "blocker_count": 1,
    "total_count": 10
  },
  "categories": [
    {
      "key": "fundamentals",
      "label": "基本面估值",
      "status": "warn",
      "coverage": 0,
      "impact": "估值、盈利质量和业务判断不可完整使用",
      "evidence": ["fundamental_snapshot 0 条"],
      "next_step": "同步财务快照和业务分部指标",
      "target_view": "fundamentals"
    }
  ],
  "next_actions": [
    {
      "priority": "P0",
      "label": "补齐主数据",
      "action": "写入 security_master",
      "target_view": "symbolWorkspace"
    }
  ]
}
```

## 状态规则

- `ready`：当前分析可使用，缺口不影响核心判断。
- `warn`：可展示，但结论必须降权或附带披露。
- `blocker`：核心判断不应生成，或会导致交易规则/执行计划错误。

第一阶段的默认口径：

- 行情缺失是 `blocker`。
- 主数据缺失是 `warn`，但交易单位缺失会在交易规则类目中体现。
- 基本面缺失是 `warn`。
- 新闻缺失或仅有占位新闻是 `warn`。
- 资金流缺失是 `warn`。
- 信号、审查、后验缺失是 `warn`，代表研究闭环未形成。
- 指数标的不要求个股基本面。

## 前端展示

个股工作台新增“分析完整度”面板：

- 总分和等级。
- ready/warn/blocker 数量。
- Top 缺口列表。
- 下一步建议。

设计风格延续当前投研工作台：高信息密度、低装饰、强调可扫描性，不做营销式大卡片。

## 测试策略

- 后端新增路由测试，覆盖快手式缺口样本。
- 测试占位新闻不会被算作真实新闻证据。
- 测试基本面/资金流/信号缺失会产生明确 next action。
- 前端通过 TypeScript build 和浏览器页面检查验证。

## 风险

- 当前工作树有大量既有未提交改动，本次只做增量，不清理其他文件。
- 第一阶段只解决“看得见缺口”，不一次性解决所有数据源接入。
- 港股真实基本面和南向/沽空数据源接入需要后续单独验证供应商稳定性。
