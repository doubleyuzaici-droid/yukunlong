# Symbol Workspace V2 — 文案审查报告

> **范围**：审查 V2 工作台所有用户可见的中文/英文文字内容（标题、占位、按钮、提示、错误、状态、空态、动态拼接的句子）。
> **不审查**：注释、变量名、log。
> **方法**：逐文件 grep + 在浏览器中模拟 5 种状态（loading / ok / partial / empty / error）逐屏对照。

### 1.2 问题诊断

这些文案的来源是**开发期为提醒自己"这块还没接"而写的占位**，没人在发布前把它们改写成面向真实用户的语言。对一个外部用户（基金研究员、个人投资者）来说：

- `BE-2` —— 不知道是什么
- `/valuation-percentile` —— 看起来像故障消息
- `akshare` —— 第三方库名，给一种"这个产品没自己的数据"的感觉
- `ETA Phase 2` —— 项目内部代号，泄露开发节奏
- "前端纯函数" —— 工程师视角，用户根本不关心是不是发请求

## 2. CTA 含义模糊（P0）

### 2.1 现状

[SymbolWorkspaceV2.tsx 主 CTA 的动态文案](TradingAgents/frontend/src/pages/symbol/SymbolWorkspaceV2.tsx)：

```typescript
return {
  label: activeTab === "pulse" ? "→ 切到图表确认"
       : activeTab === "chart" ? "→ 查看决策"
       : "→ 写入信号",
  ...
};
```

问题：每个 CTA 都是"动词 + 模糊宾语"：
- **"→ 切到图表确认"**——确认什么？确认价格？确认信号？
- **"→ 查看决策"**——谁的决策？AI 的还是研究员自己的？
- **"→ 写入信号"**——写入到哪里？这是策略动作还是数据动作？
- **"⌥ 等待回踩 EMA89 后再评估"**——这是 CTA 还是文案？看起来像建议而不是按钮

### 2.2 设计原则建议

主 CTA 文案的三条铁律：
1. **首字是明确动词**（进入/查看/创建/同步/导出/复制/分享）
2. **宾语具体**（"信号详情" 而不是 "决策"；"K 线" 而不是 "图表"）
3. **跟 decision.tone 关联**（机会 → 进场动作；风险 → 风险动作）

### 2.3 重写示例

| decision.tone | 当前 CTA | 建议 CTA |
|---|---|---|
| `opportunity` | "→ 切到图表确认" | "进入信号审查 →" 或 "查看执行清单 →" |
| `warn` | "→ 切到图表确认" | "查看证伪条件 →" |
| `risk` | "→ 切到图表确认" | "查看仓位与止损建议 →" |
| `neutral` | "→ 切到图表确认" | "进入图表寻找入场点 →" |
| `blocked`（数据缺失）| "📡 同步数据" | "同步行情数据 →" |

### 2.4 其它按钮文案

| 位置 | 现状（❌） | 建议（✅） |
|---|---|---|
| Header | "＋ 加入自选" | "加入自选" （—— 减一个特殊符号）|
| ContextRail Notes | "保存" | "保存笔记" |
| PlaybookTab | "Agent 审查" | "请 AI 审查信号" 或 "生成审查报告" |
| FundamentalsTab | "查看推理链 →" | "查看推理详情 →" |
| StatusBanner | "重试" | "重新加载" |
| Mode 切换 hint | "切到激进：买入阈值 0.55→0.45 · 仓位上限 8%→15%" | "激进模式：买入阈值更低（0.45）、单笔仓位上限更高（15%）" |

## 3. 空态文案：避免"接入中" 大集合

### 3.1 现状：5+ 处"接入中" 集中出现

| 位置 | 文案 |
|---|---|
| FundamentalsTab 估值 | "百分位接入中" |
| FundamentalsTab 财务 | "8 季度财务时间序列接入中" |
| FundamentalsTab 北向 | "北向数据接入中" |
| FundamentalsTab 龙虎榜 | "席位明细接入中" |
| CatalystTimeline | "催化剂数据接入中" |
| IndicatorMatrix 长线列 | "基本面 API ETA 2026-Q3" |
| PlaybookTab Backtest | "后验聚合接入中" |

用户连续看到 7 个"接入中"会得出一个明确结论：**"这产品做了一半就上线了"**。

### 3.2 问题诊断

空态有 5 种语义，但 V2 用同一个"接入中"模糊地说了所有：

| 真实语义 | 应该说 | 不应该说 |
|---|---|---|
| 该标的历史上没有这类数据 | "该标的暂无此项" | "接入中" |
| 后端字段还没接 | "暂未支持" 或 "即将上线" | "接入中" |
| 需要触发同步任务 | "需要先同步该数据，[同步]" | "接入中" |
| 数据落库延迟 | "数据每日 18:00 更新" | "接入中" |
| 用户层级不够（未来 SaaS） | "升级后可查看" | "接入中" |

### 3.3 重写示例

| 现状（❌） | 建议（✅） |
|---|---|
| "百分位接入中" + "等 BE-2 /valuation-percentile 路由就绪" | "估值百分位即将上线" + "联系产品获取早鸟权限 →" |
| "8 季度财务时间序列接入中" + "extend /fundamentals?quarters=8" | "8 季度财务序列即将支持" |
| "席位明细接入中" + "等 BE-7 akshare lhb_detail_em" | "暂无龙虎榜记录" + "[同步最近 30 日数据]" |
| "北向数据接入中" + "数据为累计净流入近似" | "数据来源：北向资金每日净流入累计估算（近似持股变化）"  |
| "催化剂数据接入中（后端 /api/professional/catalysts ETA Phase 2）" | "事件日历即将上线" |
| "暂无龙虎榜记录" + "数据来自 akshare · POST /api/professional/lhb/sync 同步后展示" | "暂无龙虎榜记录" + "[手动同步最新数据]" |
| "基本面 API ETA 2026-Q3" | "长线基本面指标即将支持" |

## 4. 状态/错误文案缺乏可操作性

### 4.1 现状

| 位置 | 现状（❌） |
|---|---|
| 数据状态 | "数据完整" / "部分数据缺失" / "核心数据阻断" / "数据过期" |
| Backtest | "样本不足（n < 5），结论仅供参考" |
| AsyncBoundary error | "策略服务未连接" |
| Risk Budget warn | "缺少入场参考价" |
| Falsify pill | "未触发" |
| Notes 状态 | "未保存" / "编辑中..." |

### 4.2 问题：客观描述但没有"那我怎么办"

用户看到状态文案的第一反应是"那我接下来该做什么"。V2 大量状态只说事实，不说动作。

### 4.3 重写示例

| 现状（❌） | 建议（✅） |
|---|---|
| "数据完整" | "数据已就绪" 或不显示 |
| "部分数据缺失" | "部分数据缺失，已用最新可用值替代" + 列出来源 |
| "核心数据阻断" | "核心数据缺失，决策不可靠 — [同步数据]" |
| "数据过期" | "数据过期 N 天，建议刷新 — [立即刷新]" |
| "样本不足（n < 5），结论仅供参考" | "仅 3 个历史样本，胜率统计不显著，建议作为辅助参考" |
| "策略服务未连接" | "无法连接策略服务。请确认后端运行中（默认 8100 端口），或 [查看诊断]" |
| "缺少入场参考价" | "无法估算手数：入场价缺失，请等待行情加载" |
| "未触发" | "尚未触发证伪，继续观察" |
| "未保存" | "未保存（开始输入将自动保存）" |
| "编辑中..." | "编辑中（800ms 后自动保存）" |

### 4.4 错误信息模板

建立标准格式：

```
[问题陈述]。[原因 / 建议]。[下一步动作（带按钮）]。
```

例：
> **策略服务未连接**
> 请确认本地后端运行中（默认 8100 端口）。
> [重新尝试]   [查看诊断]   [联系数据团队]

## 5. 决策理由的呈现顺序倒置

### 5.1 现状

Hero subtitle 当前模式：

> **{title}**：观察等待，趋势确认中
> **{reason}**：MACD 顶部钝化、北向连续 2 日净流出 1.1 亿，建议等待回踩 EMA89（约 ¥1,580）后再评估买点。

reason 把"理由（MACD 顶部钝化、北向）"放前面，"动作（等待回踩 ¥1580）"放后面。

### 5.2 问题：违背"读完做决定"的认知顺序

研究员扫到 Hero 时的认知流程：
1. 看 title → 判断这只票现在是什么状态
2. 想知道**接下来该做什么** → 找动作
3. 如果有时间 → 看动作的理由

把动作放前、理由放后，用户能在 2 秒内知道下一步；现在顺序反了，用户要读完整段才知道动作。

### 5.3 重写示例

| 现状（❌） | 建议（✅） |
|---|---|
| "MACD 顶部钝化、北向连续 2 日净流出 1.1 亿，建议等待回踩 EMA89（约 ¥1,580）后再评估买点。" | "**等待回踩 ¥1,580（EMA89 位置）再评估**。原因：MACD 顶部钝化、北向连续 2 日净流出 1.1 亿。" |

后端字段也可以拆开：
```typescript
decision: {
  title: "观察等待，趋势确认中",
  action: "等待回踩 ¥1,580（EMA89）",      // 新增字段
  rationale: "MACD 顶部钝化、北向连续 2 日净流出 1.1 亿",  // 重命名
  tone: "warn",
}
```

## 6. 专业术语解释不足

### 6.1 现状盘点

V2 用户界面出现以下专业缩写**未提供解释**：

| 缩写/术语 | 出现位置 | 风险 |
|---|---|---|
| RSI14 / ATR14 / KDJ-K / MACD / BOLL / VWAP | Indicator Matrix / Chart toolbar | 量化交易者认识；普通投资者陌生 |
| EMA21/89 | Indicator Matrix / Hero reason | 同上 |
| ROE TTM / PE TTM / PB / PS / EV/EBITDA | Profile Strip / Valuation Panel | 财报使用者认识；新手陌生 |
| Beta | (未来加) | 同上 |
| RS（Relative Strength） | Indicator Matrix "行业 RS" | 缩写歧义，可能误读为"做空" |
| MAE | PlaybookTab "最大不利 MAE" | Maximum Adverse Excursion，非专业不懂 |
| 量比 1.4× | Indicator Matrix | 这是 "近期成交量 / 20D 均量" 还是其它？ |
| 振幅 | (P2 缺失项) | (high - low) / prev_close 还是 / open？|
| %位 (e.g. "行业 35%位") | Profile Strip "PE TTM 行业 42%位" | 字段简略，含义不直接 |

### 6.2 解决方案：三层渐进

**Level 1（必做）**：写法变直白，减少缩写歧义
- "行业 RS" → "行业相对强弱（近 20 日）"
- "MAE" → "最大回撤"
- "量比 1.4×" → "量比 1.4×（近期均量的 1.4 倍）"
- "PE 行业 42%位" → "PE 行业百分位 42%"（明确"位"=百分位）

**Level 2（推荐）**：在缩写旁加 `title` 属性 / hover tooltip
```html
<abbr title="相对强度指数 14 日，用于判断超买/超卖">RSI14</abbr>
```

**Level 3（加分）**：在工作台底部加 "术语词典" 按钮 + Modal，专门解释指标体系

### 6.3 数字单位的中文化建议

| 现状（❌） | 建议（✅） |
|---|---|
| "5D 均收益" | "近 5 日平均收益" 或保持 "5D" 但首次出现时悬停解释 |
| "60D 均收益" | "近 60 日平均收益" |
| "近 12 月" | "过去 12 个月" |
| "20D 累计" | "近 20 个交易日平均" |
| "5,300 股" | "5,300 股（53 手）" |

| 位置 | 现状（❌） | 建议（✅） |
|---|---|---|
| PulseTab empty | "请确认 V2 策略服务可用" | "策略服务暂不可用，请稍后重试" |
| PlaybookTab empty | "当前区间未生成 V2 信号" | "当前区间未生成共振信号" |

| V1 deprecation banner | "V1 工作台已进入灰度下线流程" / "切换到 V2 →" | "旧版工作台已进入下线流程" / "切换到新版" |

### 7.2 "工作台" 这个词的多义性

文案里"工作台"出现在：
- App nav: "个股工作台"
- Right Rail eyebrow: "Research Context" → 应该有中文版
- 文档里: "工作台 V2"

建议统一：
- 用户可见名字：**"个股研究台"** 或保留 **"个股工作台"**，但**不要混用** "工作台 V2"、"V2"、"新版"
- 项目内部代号：保留 V2/SymbolWorkspaceV2 在代码层即可

### 7.3 时间表达的不一致

| 现状 | 例子 |
|---|---|
| "20D" / "60D" / "近 30 日" / "30 日" | 缩写、不缩写、加"近"、不加，四种都有 |
| "最近 12 季度" / "近 8 季度" / "8 季度" | 同上 |
| "今年" / "Q1" / "26Q1" / "2026 Q1" | 财年表达至少 4 种 |
| "延迟 3 min" / "延迟 3 分钟" | 中英文混杂 |
| "5 分钟前" / "12 分钟前" | OK |

建议统一规范：
- 时间窗口：**"近 N 日"** / **"近 N 个月"** / **"近 N 个季度"**（全部"近 N"开头）
- 季度：**"26Q1"**（财年用 2 位）
- 延迟时间：**"延迟 3 分钟"**（中文）
- 相对时间：**"X 分钟前"** / **"X 小时前"** / **"昨天 14:32"**

## 8. 句式与语气

### 8.1 风格不统一

V2 当前混用了 3 种语气：

| 语气 | 例子 |
|---|---|
| **文学化 / 偏书面** | "观察等待，趋势确认中" / "动能衰竭" / "市场环境转弱" |
| **专业 / 客观** | "EMA21/89 多头排列，趋势强度 0.82" / "MACD 顶背离持续 5 天" |
| **口语 / 偏轻** | "数据已就绪" / "评分可信" / "样本不足，仅供参考" |

第一种好，但与第三种放一起会有"高低差"。

### 8.2 建议：定一个"声音规范"

V2 应该是**沉静、可靠、克制的专业研究员**，不是热情的销售也不是冰冷的机器。

- ✅ "MACD 顶背离持续 5 天，警惕动能衰竭"（陈述事实 + 引导思考）
- ❌ "MACD 出问题了！" / "好像有点风险"（过分激情或模糊）
- ✅ "样本不足，胜率统计不显著"（坦白局限）
- ❌ "样本太少了不准的"（口语）/ "请审慎参考样本数 n < 5 的统计结果"（过分书面）

## 11. 反清单（不该做的文案优化）

❌ **不要把所有缩写都翻译成全称**
- "RSI14" 翻译成"14 日相对强度指数"会让首屏挤
- 缩写保留，hover 解释即可

❌ **不要为了"亲切"加 emoji**
- 当前 V2 已经在 V1 banner 用了 "⚠"，OK
- 但不要在文案里随便加 "📊" "📈" —— 会显得不专业

❌ **不要把"投资建议"措辞写得过软**
- "建议关注 / 可以考虑 / 似乎不错" → 没有信息量
- 用 V2 的"等待回踩 / 进入审查 / 设置提醒" 这种**具体动作**

❌ **不要做完整的多语言**
- 当前 100% 中文用户，做 i18n 是无效复杂度
- 等出海再说

❌ **不要在 UI 暴露数据精度**
- "0.829374" 这种精度对用户毫无价值
- 统一 0.83 或 82.9% 即可

## 12. 整体感受

V2 的**视觉/数据/架构都到位**，但**文案是开发者写的，不是写给用户的**。

- 工程师习惯说 "extend API"、"接入中"、"BE-2"，这些字面意义上的"待办备忘录"被直接复制到了用户界面
- CTA 用"→ 切到 X" 这种省略主语和宾语的工程缩写
- 状态描述只说事实不说动作

这是**短期内能用最低成本拉升产品质感的一块**。重写所有内部工程语言（§1）只需要 0.5 天，但用户感知会有**质变**。

完成 Sprint 文案-A + B（1 天）后，V2 的"专业品质感"会显著提升，对标买方研究终端无 gap。
