# V2 决策台界面重构设计

## 背景

当前 V2 多指标共振模块把主结论、执行清单、指标读数、买卖确认、因子拆解、价格通道和回测状态放在同一视觉层级。用户第一眼难以判断当前是否可以买、关键阻断点是什么、如果继续观察应关注哪些价格和风险条件。

本次重构采用专业交易台式分栏布局，目标是把信息阅读顺序改为：先看结论，再看决策链，再看风控和指标佐证。

## 目标

- 第一屏直接回答“现在能不能买”。
- 用 M1-M5 决策链解释“为什么”。
- 把指标和风控数值作为佐证，而不是和主结论抢层级。
- 把买入因子、卖出因子、价格计划和回测验证组织为下半区的专业工作台。
- 保持现有 V2 数据结构和策略算法不变，只重构前端呈现。

## 非目标

- 不修改后端 V2 策略算法。
- 不新增依赖。
- 不改变 API payload。
- 不引入真实交易或下单能力。
- 不重写 K 线图主体。

## 推荐信息架构

### 顶部决策台

顶部区域改为三栏：

1. **主结论栏**
   - 显示最终动作：例如推荐未通过、观察、允许试仓、风险减仓。
   - 显示策略日期、标的、模式。
   - 显示 2-3 条关键原因，优先解释阻断点。
   - 显示 S_buy、S_sell、买入确认数、卖出压力数。

2. **M1-M5 决策链栏**
   - M1 周线趋势。
   - M2 大盘过滤。
   - M3 买入触发。
   - M4 卖出风险。
   - M5 仓位约束。
   - 每一步只展示状态、关键值和一句解释。

3. **指标与风控快照栏**
   - 指标：DIF/DEA、MACD 柱、RSI14、量比、BOLL 中轨、KDJ J。
   - 风控：建议股数、建议仓位、止损价、止损距离、目标价。
   - 视觉层级低于主结论和决策链。

### 下半区专业面板

下半区保留三组模块：

1. **因子力量对比**
   - 合并买入因子和卖出因子。
   - 买入侧使用正向条，卖出侧使用风险条。
   - 用户可直接比较机会力量和风险力量。

2. **交易计划**
   - 止损价、目标价 1、目标价 2、预测高低、买回/硬止损线。
   - 建议仓位和风险比例放在同一张表中。

3. **验证状态**
   - 未运行回测时仅显示待验证状态和一句审计口径。
   - 已运行回测时显示总收益、最大回撤、胜率、交易数、成交腿、基准覆盖和 0 交易解释。

## 交互与视觉规则

- 默认展开主结论、M1-M5 决策链和交易计划。
- 指标明细、辅助买卖确认、回测审计可以作为二级信息。
- 颜色只表达状态：
  - 通过或机会：绿色。
  - 阻断或风险：红色。
  - 中性或未触发：灰色。
  - 注意或待验证：黄色。
- 避免大面积红色或绿色背景，防止页面看起来像整屏告警。
- 保持现有 8px 圆角、暗色投研台风格和紧凑信息密度。

## 组件边界

主要改动集中在：

- `TradingAgents/frontend/src/components/MarketWidgets.tsx`
- `TradingAgents/frontend/src/index.css`

建议新增局部渲染组件：

- `StrategyDecisionWorkbench`
- `StrategyDecisionChain`
- `StrategySnapshotPanel`
- `StrategyFactorComparison`
- `StrategyTradePlanCard`
- `StrategyVerificationCard`

这些组件应继续放在 `MarketWidgets.tsx` 内部，避免在本次界面整理中扩大文件结构调整。

## 测试与验证

- 运行 `npm run build` 验证 TypeScript 和 Vite 构建。
- 运行 `git diff --check` 验证无空白错误。
- 启动本地前端后检查个股工作台 V2 模块：
  - 宽屏三栏正常展示。
  - 移动端降为单栏。
  - 长中文、长数值和回测解释不溢出。
  - 主结论和 M1-M5 阻断点清晰可读。

## 风险

- `MarketWidgets.tsx` 当前改动较多且处于未跟踪状态，重构时需要保留现有 K 线和策略控制逻辑。
- 如果 API 某些字段为空，组件必须继续优雅降级显示 `-` 或待验证状态。
- 专业交易台式布局会更强烈地强调“动作建议”，文案仍需保持研究辅助属性，避免暗示真实交易指令。
